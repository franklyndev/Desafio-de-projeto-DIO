Pra que serve o Git?

Git <> Github = são diferentes

Git: software de controle de versão

Github: Plataforma de rede social pra programadores para postar seus repositórios

Git: Software de controle de versão(VCS) ou versionamento, chamamos o git dessa maneira porque ele é como se fosse uma máquina do tempo que guarda as versões do seu projeto. Imagine que está criando um site pra um cliente e aí vc termina de fazer o html, aí vc salva em uma pasta como site-cliente, ao consultar seu cliente ele diz que tá ficando top, aí vc guarda essa pasta numa pasta zip, pra se acontecer alguma bosta no seu projeto no futuro, vc voltar nessa pasta e fazer de novo. Mas a cada vez que você avança no projeto é necessário salva-lo muitas vezes, nesse pique vc vai acumulando muitas pastas zip e no futuro isso vai ficar uma bagunça, pois tem muitas VERSÕES do seu projeto misturadas. por isso existe o Git que é Software de controle de versão(VCS) ou versionamento.

Git é um VCS distribuído que funciona da seguinte forma: Repositório remoto(GitHub) < push < repositório local(GIT) < commit < programador

![image.png](attachment:845f8c32-11e9-41f9-92da-bae9253f4627:image.png)

Commit: É um registro da versão do estado do projeto atual, que vc manda para o git, é como um salvamento do que vc tava fazendo agr que vc pode mexer nele depois.

Vantagens do VCS: 

- Consulta de histórico - Voltar da onde vc parou
- Ramificação - Trabalho em equipe, cada um faz sua parte e depois juntar fazendo o projeto principal.
- Organização

GitHub - É um site onde vc guarda seus repositórios e compartilha com outros , como se fosse uma rede social de programadores, lá vc guarda projetos.

- Forks - Você se tornar colaborador de um projeto e ajudar nele
- Um **fork** no GitHub é uma cópia de um repositório existente que você cria em sua própria conta. Isso permite que você faça alterações no seu repositório bifurcado sem afetar o repositório original.
